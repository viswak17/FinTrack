### Complete Project Prompt v2.1

## 🌌 Project Philosophy

> **"Money shouldn't weigh you down."**

FinTrack is not an expense tracker. It is a **Personal Finance Operating System** — a command center where every rupee, goal, subscription, and financial decision is visible, intelligent, and in your control. Where the original tracker reacted to your data, FinTrack **thinks ahead of it** — using real machine learning models trained on your own financial patterns.

The experience should feel like a Bloomberg Terminal crossed with a luxury fintech app — powerful, fast, and deeply satisfying to use.

**Product Name:** FinTrack  
**Tagline:** *Track smarter. Spend wiser. Own your money.*  
**Type:** Personal Expense Tracker (Web Application)

### 9.3 ANOMALY DETECTION

```
ML Model:       Isolation Forest + Z-score (dual method)
Library:        scikit-learn (IsolationForest)
Input:          Amount + category + day-of-week + time features

Z-score:        Per-category threshold (flags if > 2.5 std from category mean)
```

### 9.4 SPENDING PATTERN ANALYSIS

```
ML Models:
  ├── K-Means Clustering (scikit-learn) — spending behavior clusters
  ├── Rolling statistics (pandas) — trend detection
  └── Pearson correlation — category relationship discovery

Features produced:

A) Behavioral Clusters:
   User's spending grouped into patterns:
   e.g., "Weekend Spender", "Subscription Heavy", "Daily Coffee Habit"
   → Shown as a user profile badge on AI Advisor page

B) Category Trend Detection:
   Per-category rolling 4-week trend (increasing / stable / decreasing)
   e.g., "Food spend trending UP +18% over last 4 weeks"
   Algorithm: Slope of linear regression on weekly category totals

C) Category Correlations:
   e.g., "When you spend on Travel, Entertainment also spikes (+67%)"
   Algorithm: Pearson correlation matrix across category weekly totals
   → Shown as ECharts HeatMap chart on AI Advisor

D) Day-of-Week Analysis:
   Which days you spend most/least
   Algorithm: groupby(day_of_week).mean() on transaction amounts
   → Shown as ECharts Radar chart (7-axis, one per day)

E) Recurring Detection (unsupervised):
   Detects transactions that appear regularly even if not manually flagged
   Algorithm: FFT (Fast Fourier Transform via numpy) on transaction sequences
              + DBSCAN clustering on (amount, day_of_month) pairs
   → "Detected: ₹649 appears every ~30 days — possible subscription"
```

### 9.6 BUDGET RECOMMENDATION ENGINE

```
Algorithm:     Descriptive statistics (pandas) + Percentile-based suggestion
Logic:
  For each category with 3+ months of history:
    mean_spend = rolling 3-month average
    std_spend  = rolling 3-month std deviation
    suggested  = mean_spend + (0.5 * std_spend)   ← slightly above average
    tight      = mean_spend                         ← exactly at average
    relaxed    = mean_spend + std_spend             ← comfortable buffer

UI shows 3 options (Tight / Balanced / Relaxed) with ML-computed amounts
```

```
  ├── AI Chat Advisor (open-ended conversation)
  ├── Converting ML outputs → natural language pulse bullets
  └── Generating export-ready monthly narrative summaries
```

**Groq API Call Pattern:**
```python
async def get_llm_response(prompt: str, system: str) -> str:
    response = await groq_client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[
            {"role": "system", "content": system},
            {"role": "user",   "content": prompt}
        ],
        max_tokens=600,
        temperature=0.4
    )
    return response.choices[0].message.content
```

**Chat Advisor Examples:**
```

Cold Start Strategy:
  New user (< 20 transactions):
```

## 📊 Full Data Flow

```
User adds "₹450 Swiggy":
 1. NLP parser extracts: amount=450, payee=Swiggy, date=today
 2. ML Categorizer: suggests "Food Delivery" (94% confidence)
 3. User confirms → transaction saved (account balance -= 450)
 4. Anomaly Detector runs: is ₹450 unusual for Swiggy? → No flag
 5. Budget updated (Food: spent += 450)
 6. If Food budget > 80% → notification + ML pace alert recalculates
 7. Recurring detector: does this match a recurring pattern?
 8. Goal check: auto-contribution if linked
 9. Prophet model: adds data point, forecast refreshes (next refit: Sunday)
10. AI Pulse cache invalidated → next dashboard load regenerates
11. All 15 report datasets update on next fetch
```
